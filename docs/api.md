# Спецификация REST API

Базовый префикс: `/api`. Авторизация — JWT в заголовке
`Authorization: Bearer <token>`. Все запросы и ответы — JSON в кодировке UTF-8.

## Коды ответов

| Код | Когда                                                                |
|-----|----------------------------------------------------------------------|
| 200 | Успех (GET/PUT/PATCH)                                                |
| 201 | Создан новый ресурс (POST)                                           |
| 204 | Успех без тела (DELETE)                                              |
| 400 | Невалидное тело запроса или нарушено бизнес-правило                  |
| 401 | Отсутствует/невалидный токен                                         |
| 403 | Авторизация есть, но не хватает прав (например, не админ)            |
| 404 | Сущность не найдена                                                  |
| 422 | Не прошла валидация Pydantic (формат полей, длины, диапазоны)        |

## Аутентификация

### `POST /api/auth/register`

Регистрирует нового сотрудника и сразу возвращает access-токен.

Запрос:

```json
{
  "email": "ivanov@neftegaz.ru",
  "full_name": "Иванов Иван",
  "password": "qwerty123"
}
```

Ответ `201`:

```json
{
  "access_token": "eyJhbGciOi...",
  "token_type": "bearer",
  "user": {
    "id": 5,
    "email": "ivanov@neftegaz.ru",
    "full_name": "Иванов Иван",
    "role": "employee",
    "is_active": true,
    "created_at": "2026-05-28T10:00:00Z"
  }
}
```

### `POST /api/auth/login`

Запрос:

```json
{ "email": "admin@neftegaz.ru", "password": "admin123" }
```

Ответ аналогичен `register`. Ошибка `401` при неверном пароле.

### `GET /api/auth/me`

Требует токен. Возвращает текущего пользователя (`UserOut`).

## Месторождения (`/api/fields`)

| Метод   | URL                       | Кто может         | Описание                                       |
|---------|---------------------------|-------------------|------------------------------------------------|
| GET     | `/api/fields`             | Все               | Список с фильтрами `q`, `status`               |
| GET     | `/api/fields/{id}`        | Все               | Карточка с числом скважин (`wells_count`)      |
| POST    | `/api/fields`             | Admin             | Создать месторождение                          |
| PUT     | `/api/fields/{id}`        | Admin             | Обновить (частично)                            |
| DELETE  | `/api/fields/{id}`        | Admin             | Удалить (каскадно с скважинами и заявками)     |

Пример тела для создания:

```json
{
  "name": "Уренгойское",
  "location": "ЯНАО",
  "reserves_tons": 850000,
  "discovered_year": 1966,
  "status": "active",
  "description": "Крупное газоконденсатное месторождение"
}
```

## Скважины (`/api/wells`)

| Метод   | URL                       | Кто может | Описание                                     |
|---------|---------------------------|-----------|----------------------------------------------|
| GET     | `/api/wells`              | Все       | Фильтры: `field_id`, `status`                |
| GET     | `/api/wells/{id}`         | Все       | Карточка скважины                            |
| POST    | `/api/wells`              | Admin     | Создать (требует существующего месторождения)|
| PUT     | `/api/wells/{id}`         | Admin     | Обновить                                     |
| DELETE  | `/api/wells/{id}`         | Admin     | Удалить                                      |

## Заявки на обслуживание (`/api/requests`)

| Метод   | URL                          | Кто может                              | Описание                              |
|---------|------------------------------|----------------------------------------|---------------------------------------|
| GET     | `/api/requests`              | Авторизованные                         | Фильтры: `well_id`, `status`, `priority` |
| POST    | `/api/requests`              | Авторизованные                         | Создать (автор = текущий пользователь)|
| PATCH   | `/api/requests/{id}`         | Автор или admin                        | Изменить статус, приоритет, бригаду   |
| DELETE  | `/api/requests/{id}`         | Автор или admin                        | Удалить                               |

Возможные значения `status`: `new` → `in_progress` → `done` / `rejected`.
Поле `closed_at` заполняется при переводе в `done`.

| PATCH   | `/api/brigades/{id}`       | Admin             | Обновить                            |
| DELETE  | `/api/brigades/{id}`       | Admin             | Удалить                             |

## Пользователи (`/api/users`)

| Метод   | URL                       | Кто может | Описание                                |
|---------|---------------------------|-----------|-----------------------------------------|
| GET     | `/api/users`              | Admin     | Список всех пользователей               |
| PATCH   | `/api/users/{id}`         | Admin     | Изменить роль / активность              |
| DELETE  | `/api/users/{id}`         | Admin     | Удалить пользователя                    |

## Статистика (`/api/stats`)

### `GET /api/stats/overview`

Доступно авторизованным. Возвращает агрегаты по системе:

```json
{
  "fields_total": 5,
  "fields_active": 3,
  "wells_total": 12,
  "wells_operating": 9,
  "requests_open": 4,
  "requests_done": 8,
  "daily_output_tons": 1230.5,
  "brigades_available": 2
}
```

## Технические эндпоинты

- `GET /api/health` — проверка живости сервиса.
- `GET /docs` — Swagger UI (FastAPI).
- `GET /redoc` — ReDoc.
- `GET /openapi.json` — машинная спецификация OpenAPI.
